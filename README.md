# AceStream Mac Player

Простой нативный macOS-плеер для открытия `acestream://...` ссылок через локальный AceStream Engine.

## Как это работает

AceStream-ссылка сама по себе не является обычным видеофайлом. Для нее нужен AceStream Engine, который поднимает локальный HTTP-поток. Приложение берет ссылку вида:

```text
acestream://0123456789abcdef0123456789abcdef01234567
```

и открывает ее как:

```text
http://127.0.0.1:6878/ace/getstream?id=0123456789abcdef0123456789abcdef01234567
```

Этот поток воспроизводится встроенным macOS AVKit-плеером.

## Сборка

Нужны Xcode Command Line Tools:

```bash
xcode-select --install
```

Собрать приложение:

```bash
./build.sh
```

Готовое приложение появится здесь:

```text
build/AceStream Mac Player.app
```

## Быстрый запуск

1. Установите Docker Desktop для Mac: https://www.docker.com/products/docker-desktop/
2. Откройте `build/AceStream Mac Player.app`.
3. Вставьте `acestream://...` ссылку и нажмите `Открыть`.

Приложение само попробует запустить Docker Desktop и контейнер AceStream Engine. Первый запуск может занять несколько минут, потому что Docker скачает образ `jopsis/aceserve`.

Для встроенного macOS-плеера AceStream-поток автоматически преобразуется в локальный HLS через ffmpeg-контейнер `lscr.io/linuxserver/ffmpeg` и отдается приложением через локальный HTTP-сервер. Этот образ тоже скачивается один раз при первом запуске.

Если хотите запустить все из терминала:

```bash
./scripts/run-player.sh
```

Скрипт запустит AceStream Engine в Docker, соберет приложение при необходимости и откроет плеер.

## Использование

1. Откройте `build/AceStream Mac Player.app`.
2. При необходимости нажмите `Запустить Engine`, или просто вставьте ссылку и нажмите `Открыть`.
3. Вставьте `acestream://...`, 40-символьный content id или прямой `http(s)` URL.
4. Нажмите `Открыть`.

Если engine запущен на другом адресе, поменяйте поле `Engine` в приложении.

## Если ссылка не запускается

Нажмите `Проверить` рядом с полем `Engine`. Если приложение пишет, что Engine не отвечает, значит на Mac не запущен локальный AceStream Engine и видео-поток еще не существует.

Проверить это можно и из терминала:

```bash
curl -I 'http://127.0.0.1:6878/ace/getstream?id=aedf221240a0f8513d90b555bf1b6e5306ac1e76'
```

Если ответ `Couldn't connect to server`, сначала нужно запустить AceStream Engine, а уже потом открывать `acestream://...` ссылку в плеере.

Если приложение пишет, что трансляция не отдает данные, Engine запущен, но конкретная AceStream-ссылка сейчас не имеет live-данных: источник может быть выключен, ссылка устарела или нет peers.

## Ограничение

Это не полноценная реализация AceStream-протокола. Приложение является легким macOS-плеером/оболочкой и ожидает работающий AceStream Engine. Реализовать сам P2P-протокол AceStream без официального engine нельзя надежно: это отдельный закрытый сетевой слой, а не формат видео.
